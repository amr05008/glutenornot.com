## Summary
Built and uploaded iOS v1.1.1 (build 1) to App Store Connect. Release bundles the Sentry `beforeSend` filter for `not_found` barcode errors, server-side barcode miss logging, and the client-side duplicate scan cache from the 2026-04-14 Sentry audit.

## Changes
- `mobile/app.json`: version 1.1.0 → 1.1.1
- Regenerated `mobile/ios/` via `npx expo prebuild --platform ios --clean` (gitignored)
- Manually patched `MARKETING_VERSION` in `ios/GlutenOrNot.xcodeproj/project.pbxproj` from `1.0` → `1.1.1` (both Debug and Release) — prebuild drops the patch segment
- Added `SENTRY_AUTH_TOKEN` export to `~/.zshrc` so future prebuilds upload source maps

## Decisions
- Re-ran prebuild after setting `SENTRY_AUTH_TOKEN` so source maps uploaded during this build rather than shipping without them.
- Build number stayed at 1 (first build for the 1.1.1 version string, which is valid since `CFBundleVersion` is scoped per `CFBundleShortVersionString`).

## Notes
- Known quirk: `expo prebuild` sets `MARKETING_VERSION = 1.0` — must be patched after every prebuild.
- Post-release manual tasks (browser-only):
  - Verify Vercel production env has `USDA_API_KEY`, `NUTRITIONIX_APP_ID`, `NUTRITIONIX_API_KEY`
  - Create Sentry alert rules on `glutenornot-mobile` for (1) new error-level issues, (2) warning-level event spikes
