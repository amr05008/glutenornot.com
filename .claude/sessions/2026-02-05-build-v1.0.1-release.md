## Summary
Built and submitted v1.0.1 to the App Store. This release includes three fixes committed since v1.0.0: Sentry crash reporting, background scan hang fix, and camera crash fix.

## Changes
- `mobile/app.json` — bumped `expo.version` from `1.0.0` to `1.0.1`

## What's in v1.0.1
- Fixed camera crash when capturing before session is ready (`07b687a`)
- Fixed app hanging on scan after prolonged background use (`748fceb`)
- Added Sentry crash reporting for improved reliability (`9d2292a`)

## Build Details
- EAS production build with `autoIncrement: true` (build number 4)
- Distribution certificate reused from v1.0.0
- New provisioning profile generated (Developer Portal ID: 34LSHXFVHP)
- SENTRY_AUTH_TOKEN EAS secret confirmed for source map uploads
- Submitted to App Store Connect via `eas submit --platform ios --latest`

## Notes
- `eas build:version:set` requires interactive input; can't be run non-interactively
- `eas build --non-interactive` fails if credentials need validation — must run interactively
- Free tier build queue had ~200 min wait time
